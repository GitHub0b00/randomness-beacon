Intro.

Click "The Beacon" button in the navigation bar to access the beacon. Each page on the display you see is named as a "pulse". The data formating and hash chain connection follows the NIST Randomness Beacon Reference [1].

Data Fields

localRandomValue
The random string generated in the previous minute.
precommitmentValue
The hash of localRandomValue of the next pulse.
uri
The uri linked to the corresponding JSON data. It can be used as an API request to be integrated into your service too.
pulseIndex
The integer index of the beacon pulse.
chainIndex
The integer index of the beacon pulse.
timeStamp
The approximate time that this pulse is generated.
previous
The previous pulse.
hour
The outputValue of the first pulse in the hour that the "previous" pulse belongs to.
day
The outputValue of the first pulse in the day that the "previous" pulse belongs to.
month
The outputValue of the first pulse in the month that the "previous" pulse belongs to.
year
The outputValue of the first pulse in the year that the "previous" pulse belongs to.
signatureValue
The concatenation of all the above fields signed with a private key.
outputValue
The hash output of the concatenation of all the above fields.

Highlights

Pulses are connected to each other by means of input and output of the hash function. On the one hand, "previous", "hour", "day", "month", and "year" stores the outputValue of the corresponding "previous", "hour", "day", and "month" pulses, where the "outputValue" values are themselves hash of all the other fields' values. On the other hand, "precommitmentValue" stores the hash of the "localRandomValue" of the next pulse.
